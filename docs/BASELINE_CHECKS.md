# Phase 0: Baseline Checks

Цель: зафиксировать проверяемый baseline до рефакторинга, чтобы на каждом этапе подтверждать, что текущее поведение не сломано.

## Быстрый smoke

Запуск из корня проекта:

```bash
bash scripts/smoke_baseline.sh
```

Скрипт проверяет:

- Python-модули в `src/` компилируются
- CLI доступен (`python3 -m src.main --help`)
- `config/sources.json` валиден и содержит уникальные `source.id`
- JSON-файлы в `config/`, `state/`, `data/` валидны

Важно: это offline/safe проверки, без сетевого краулинга.

## Ручная функциональная проверка (после каждого этапа)

1. Проверить, что интерфейс запуска не изменился:

```bash
python3 -m src.main --help
```

2. Проверить точечный запуск источника (если сеть доступна):

```bash
python3 -m src.main --sources myfonts --timeout 20
```

3. Проверить бэкфилл-режим (если сеть доступна):

```bash
python3 -m src.main --history-weeks 1
```

4. Проверить фронтенд:

```bash
cd web
npm run dev
```

Открыть `http://localhost:3000` и убедиться, что:

- список релизов загружается
- блок `Sources` отображается
- карточки/группировка по неделям отображаются без ошибок

## Критерии "этап не сломал baseline"

- `scripts/smoke_baseline.sh` проходит без ошибок
- CLI флаги работают как до этапа
- фронтенд поднимается и читает текущие данные
- (опционально) на сетевом прогоне не появилось новых ошибок в консоли для тех же источников
