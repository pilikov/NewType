# Куда делись данные января (MyFonts)

## Текущее состояние (после бэкапа 2026-03-10)

- **Неделя 2–8 марта:** ~291 релиз MyFonts — всё в порядке.
- **Неделя 5–11 января:** в выдаче только 4 релиза MyFonts.

## Откуда берутся данные

1. **Day-снимки** — `web/data/myfonts/2026-03-07`, `2026-03-08`, `2026-03-09`, `2026-03-10`. Берётся только **последний** (`findLatestDayDir` → `2026-03-10`).
2. **Периоды** — все каталоги в `web/data/myfonts/periods/`:
   - `2025-12-28_2026-03-07` — покрывает январь, но в файле **всего 2 релиза** (даты 2026-02-08 и 2026-02-11).
   - `2026-02-16_2026-02-22` — 10 релизов.
   - `2026-02-16_2026-03-08` — 1012 релизов (февраль–март).

Данные объединяются: сначала day, потом все периоды; дедуп по семье, приоритет у day. Недели строятся по полю даты релиза (`myfonts_debut_date` / `release_date`).

## Почему мало января

1. **Период 2025-12-28_2026-03-07** в репозитории изначально содержал только 2 релиза (проверено по коммиту `5ca5cd25` "Add web data snapshot"). То есть полноценного снимка за декабрь–январь в репо никогда не было.
2. **Day-снимки** (2026-03-09, 2026-03-10) в сырых данных содержат по **4 релиза** с датой в неделе 5–11 января (2026-01-06, 08, 09). Остальные январские даты там в основном 26–30 января (много релизов). То есть в текущих снимках неделя 5–11 января изначально представлена только этими 4 релизами.

Вывод: **74 релиза за 5–11 января в текущих данных нигде не сохранялись** — либо это была оценка/другой источник, либо снимок за тот период не попадал в репо (другой диск, старый краул и т.д.).

## Как вернуть январь

Чтобы снова иметь нормальное покрытие 5–11 января:

1. Запустить полный краул MyFonts за период, включающий январь (например, 2025-12-28 — 2026-03-07), и сохранить результат в `web/data/myfonts/periods/2025-12-28_2026-03-07/`.
2. Либо восстановить такой снимок из своего архива/бэкапа, если он есть вне репозитория.

Текущая логика загрузки (day + все периоды, дедуп по семье, фильтр по ссылке на семью) менять не нужно — она уже подтягивает все имеющиеся периоды, включая `2025-12-28_2026-03-07`.
